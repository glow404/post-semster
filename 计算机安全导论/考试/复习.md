## 第三章

信息安全的基本目标：**保密性、完整性、可用性、可控性**

信息安全的发展历程：物理安全、网络安全、应用安全、数据安全

信息安全的主要威胁（人为失误，信息截取，内部窃密破坏，黑客攻击，技术缺陷，病毒），安全威胁的主要表现特征：窃听，重传，伪造，篡改，拒绝服务攻击，行为否认，非授权访问，传播病毒。



攻击者攻击的主要对象：网络基础设施，公共互联网，新兴信息技术

社会工程学攻击是攻击者通过利用人的心理弱点和行为习惯，诱骗合法用户泄露敏感信息或执行非法操作的一种攻击方式。

攻击形式包括：钓鱼，下饵，冒充（假托）等等



**PDRR 模型**是一种**信息安全管理与防护的整体框架模型**

**PDRR** 分别代表：

- **P — Protection（防护）**
- **D — Detection（检测）**
- **R — Response（响应）**
- **R — Recovery（恢复）**

# 二、美国 TCSEC 评测准则（P130–132）

## 1️⃣ 定义

**TCSEC（Trusted Computer System Evaluation Criteria）**

- 又称 **橙皮书（Orange Book）**
- 是美国国防部制定的 **计算机系统安全评价标准**
- 对系统安全功能和可信度进行分级

------

## 2️⃣ 安全等级分类（D ~ A 类）

### D 类：未评价系统

- 系统不满足基本安全要求
- 无安全保障

### C 类：自主保护系统（Discretionary Protection）

- **C1（Discretionary Security Protection）**
  - 提供最基本的访问控制
  - 用户能自行控制对对象的访问
- **C2（Controlled Access Protection）**
  - 更严格的访问控制
  - 用户身份认证更完整
  - 审计日志记录安全事件

### B 类：强制保护系统（Mandatory Protection）

- **B1（Labeled Security Protection）**
  - 文件和数据带有安全标签
  - 强制访问控制（MAC）引入
  - 支持有限的审计
- **B2（Structured Protection）**
  - 系统结构化设计
  - 提供安全内核和更严格的访问控制
  - 审计和身份认证增强
- **B3（Security Domains）**
  - 高度可信系统
  - 安全策略强制执行
  - 防止安全策略被绕过

### A 类：验证保护系统（Verified Protection）

- **A1（Verified Design）**
  - 最高安全等级
  - 系统经过形式化设计验证
  - MAC、审计、身份认证全面实现
  - 可提供**最高级别机密性保障**

PE 文件型病毒要点

| 重点项                                | 必背要点                                                     |
| ------------------------------------- | ------------------------------------------------------------ |
| 文件感染的核心要求                    | 让目标PE具备病毒功能，但不破坏宿主原功能与外在形态（例如图标）。 |
| 感染为什么容易“翻车/暴露”             | 病毒插入宿主不同位置会迫使修改PE头相关参数；改错会导致宿主异常从而暴露。 |
| Win32 运行/夺权总流程（顺序要会写）   | 用户触发HOST→装载到内存→通过**AddressOfEntryPoint + ImageBase**定位第一条指令→执行（病毒可能在入口或运行中夺权）→病毒执行后归还控制权→HOST继续运行。 |
| AddressOfEntryPoint（AEP）            | PE装载器准备运行的第一条指令RVA；属于病毒感染的关键字段。    |
| ImageBase                             | PE优先装载地址；入口定位常用AEP+ImageBase；但发生重定位时ImageBase不一定等于真实装载位置。 |
| 常见感染方式（控制权交换）            | 通过改入口点AEP或EPO等方式夺权；需记录原控制点，病毒执行后归还；通常还会设置感染标记避免重复感染。 |
| 感染对象与遍历搜索                    | 常感染PE格式文件（EXE/SCR/DLL等）；全盘/部分盘符遍历常用 FindFirstFile、FindNextFile。 |
| 病毒为何需要重定位                    | 寄生位置不固定；DLL加载位置/注入环境不确定，因此需要重定位；课件给出利用 Call 语句进行重定位思路。 |
| 为何“自获取API函数”（不走宿主导入表） | 直接改host导入节会引起结构变化、修正成本高，甚至破坏原功能；因此病毒倾向自行解析API。 |
| 自获取API的关键API与套路              | 关键API：GetProcAddress、LoadLibraryA；套路：先找 kernel32.dll 模块基址，再通过导出表定位函数地址。 |
| 捆绑释放型病毒（典型：熊猫烧香）      | 病毒体内“携带HOST数据”：运行时先执行病毒，再还原执行HOST；优点实现简单效率高、可感染自校验程序；缺点主体是病毒更易被发现且图标等异常。 |
| 系统感染型病毒（扩展常考点）          | 不感染PE文件、目标是操作系统；两关键问题：再次获得控制权（自启动）与传播；自启动手段包括注册表键值、特定位置/文件、Autorun.inf、嵌入触发代码、DLL劫持等。 |
