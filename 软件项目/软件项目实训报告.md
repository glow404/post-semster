## 2.1 任务描述

### 2.1.1 项目背景

传统指纹识别方法（如基于细节点 Minutiae）在以下场景存在局限：

- 图像质量较差（噪声、模糊）
- 指纹旋转、平移、尺度变化
- 特征提取依赖人工规则

为提升鲁棒性，本项目引入**深度学习方法——孪生网络（Siamese Network）**，通过学习指纹图像的**特征嵌入（Embedding）**，实现端到端的相似度度量。

------

### 2.1.2 项目目标

构建一个基于孪生网络的指纹识别系统，实现：

- 输入两张指纹图像，输出相似度（0~1）
- 支持1:1验证（Verification）与1:N识别（Identification）
- 提供可扩展的深度学习框架

------

### 2.1.3 核心任务分解

| 模块       | 任务内容                        |
| ---------- | ------------------------------- |
| 数据模块   | 数据集构建、正负样本对生成      |
| 预处理模块 | 图像归一化、裁剪、增强          |
| 模型模块   | Siamese网络结构设计             |
| 损失函数   | Contrastive Loss / Triplet Loss |
| 训练模块   | 模型训练与调参                  |
| 推理模块   | 相似度计算与阈值判定            |
| 系统集成   | API或界面封装                   |

------

### 2.1.4 输入输出定义

- 输入：
  - 指纹图像对 (Image A, Image B)
- 输出：
  - 相似度分数 ( S )
  - 匹配结果（Match / Non-Match）

#### （1）模型可行性

孪生网络结构：

- 两个共享权重的CNN
- 输出固定维度Embedding（如128维）
- 使用距离函数（Euclidean / Cosine）衡量相似度

优势：

- 对小样本数据友好
- 泛化能力较强

## 2.4 设计报告

### 2.4.1 系统总体架构

```text
数据层 → 预处理层 → 特征提取层 → 相似度计算 → 判定层
```

------

### 2.4.2 孪生网络结构设计

#### 网络结构

```text
        ┌──────── CNN (shared weights) ────────┐
Input A ─►                                      ├─► Embedding A ─┐
        └──────────────────────────────────────┘                │
                                                               ├─ Distance ─► Output
        ┌──────── CNN (shared weights) ────────┐                │
Input B ─►                                      ├─► Embedding B ─┘
        └──────────────────────────────────────┘
         ┌─ CNN ─ Embedding B ─┘
```

- 权重共享（Weight Sharing）
- 输出向量维度：128/256

------

### 2.4.3 损失函数设计

采用**对比损失（Contrastive Loss）**：
$$
L = (1 - Y) \cdot D^2 + Y \cdot \max(0, m - D)^2
$$
说明：

- (Y=0)：同类（匹配）
- (Y=1)：不同类
- (D)：Embedding距离
- (m)：间隔参数（margin）

------

### 2.4.4 相似度计算

- 欧氏距离：
  $$
  [
  D = ||f(x_1) - f(x_2)||
  ]
  $$
  

- 余弦相似度：
  $$
  [
  S = \frac{f(x_1)\cdot f(x_2)}{||f(x_1)|| ||f(x_2)||}
  ]
  $$
  

------

### 2.4.5 数据处理流程

1. 图像归一化（大小统一）
2. 数据增强：
   - 旋转
   - 加噪声
   - 随机裁剪
3. 构造正负样本对

### 3.1 实训收获

这次实训是我第一次比较完整地接触**基于深度学习的指纹识别系统**，相比以前做过的一些传统算法实验，整体收获还是比较明显的。

首先是在**理论理解方面**。通过实现孪生网络，我对“特征学习”和“相似度度量”有了更直观的认识。以前更多是用人工设计特征，而这次是让模型自己去学习指纹的表示（embedding），这种思路上的转变对我影响比较大。同时，对对比损失函数的作用也有了更具体的理解，不再只是停留在公式层面。

其次是在**工程实践方面**。这次基本是按照一个完整项目流程来做的，从最开始的数据准备、模型搭建，到后面的训练、测试和调参，整个过程走了一遍。对PyTorch的使用也更熟练了一些，比如数据加载、模型训练流程、模型保存等。相比只写单个实验代码，这种完整项目更能锻炼实际开发能力。

另外，在**问题分析能力**上也有提升。过程中遇到过模型不收敛、效果不稳定等问题，一开始比较盲目，后来通过查资料、调整参数、对比实验，逐步找到原因。这种不断试错和调整的过程，对理解模型其实帮助很大。

------

### 3.2 实训经验

这次实训中总结了一些比较实际的经验：

**（1）数据比模型更重要**

一开始我比较关注模型结构，但后面发现，数据质量对结果影响更大。比如指纹图像清晰度、是否做了合理的数据增强，都会直接影响模型效果。适当的数据增强（旋转、加噪声）对提升泛化能力也很有帮助。

------

**（2）样本构造很关键**

孪生网络不是直接用单张图片训练，而是用“样本对”。如果正样本和负样本构造得不好，比如太简单或者不均衡，模型就很难学到有效特征。这一点在实际训练中体会比较明显。

------

**（3）参数和阈值需要调试**

模型训练好之后，并不是直接就能用，还需要调整相似度阈值。不同阈值下，误识率和拒识率差别很大，所以需要结合测试结果进行调整，而不是随便设一个值。

------

**（4）分模块开发更高效**

在实现过程中，把系统分成数据处理、模型、测试几个模块之后，调试起来方便很多，也更容易修改和扩展，比如替换网络结构时影响范围比较小。

------

### 3.3 存在不足

这次实训也暴露出一些问题：

- **模型还比较简单**：使用的是较基础的CNN结构，没有尝试更复杂或更成熟的模型（如ResNet）
- **数据量不够大**：训练数据有限，对一些复杂情况（模糊、部分缺失）效果不太好
- **调参不够系统**：很多参数调整还是靠经验，没有形成系统的方法
- **工程化不足**：目前只是实验性质的代码，没有做成完整系统（比如界面或接口）

------

### 3.4 改进建议

如果后续继续完善这个项目，可以从以下几个方面改进：

- 在**模型方面**，尝试更深的网络结构，或者引入Triplet Loss等方法
- 在**数据方面**，增加数据量，同时加强数据增强
- 在**系统方面**，可以做一个简单的界面或者Web接口，让系统更完整
- 在**性能方面**，可以考虑优化匹配速度，比如使用向量检索方法

------

从课程角度来说，有几点建议：
